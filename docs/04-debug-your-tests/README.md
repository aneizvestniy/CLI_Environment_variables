# 04 — Debug your tests

## Мета модуля

Навчитись швидко знаходити причину падіння тесту:
- IDE debug (VS Code / WebStorm)
- Playwright Inspector (`--debug`)
- `page.pause()`
- робота з flaky тестами

## Основні прийоми

- Спочатку зроби **мінімальний reproduction** (один тест / один файл).
- Використовуй `--headed` + `--debug` для UI проблем.
- Дивись на **trace** і HTML report.
