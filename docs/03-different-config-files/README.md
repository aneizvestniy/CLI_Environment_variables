# 03 — Different config files

## Мета модуля

Зрозуміти, як керувати різними конфігураціями Playwright і **не заплутатись**:

- що таке `playwright.config.*`
- коли робити окремі конфіг-файли
- як запускати через `--config`
- коли краще залишити **один** конфіг і використовувати `projects`

## Пояснення для новачків

### Що таке конфіг Playwright

Файл `playwright.config.ts` — це “налаштування запуску тестів”. Він визначає:
- таймаути
- репортер
- браузери (projects)
- baseURL
- headless/headed
- trace/video/screenshot

### Чому інколи роблять кілька конфігів

Якщо різні набори тестів мають **дуже різні** потреби, зручніше мати 2 конфіги, ніж один "монстр".

Приклади:
- API тести: без браузера, швидкі
- E2E тести: з браузером, довгі таймаути

## Коли робити окремі конфіги

- у вас сильно різні типи тестів (API vs E2E)
- різні репортери/таймаути
- різні baseURL/credentials

## Коли краще один конфіг + projects

- різні браузери
- різні теги (`@smoke`, `@regression`)
- різні середовища (через env vars)
