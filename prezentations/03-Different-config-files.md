# Different config files

Використання декількох конфігурацій у Playwright дозволяє створювати різні набори налаштувань для вашого тестового середовища та виконання тестів у різних умовах.

## Навіщо потрібні декілька конфігів

Ось типові сценарії:

- **Різні середовища**: окремі конфіги для local / stage / prod.
- **Розділення тестів**: наприклад, один конфіг для API тестів, інший — для E2E.
- **Кросбраузерні тести**: різні браузери (Chromium, Firefox, WebKit) можуть мати різні налаштування.
- **Різні умови тестування**: headed/headless, різні таймаути, viewport, тощо.
- **Тестування різних версій додатку**: конфіги для різних гілок/релізів.

---

## Приклад: окремий конфіг для E2E

Створимо файл `playwright.config.e2e.js`:

```js
// playwright.config.e2e.js
module.exports = {
  projects: [
    {
      name: 'E2E tests',
      use: {
        baseURL: 'https://www.example.com',
        contextOptions: {
          ignoreHTTPSErrors: false,
          viewport: { width: 1920, height: 1080 },
        },
      },
    },
  ],
};
```

Щоб Playwright використав цей конфіг файл:

```bash
npx playwright test --config=playwright.config.e2e.js
```

---

## Важлива примітка

У більшості реальних проєктів можна обійтися **одним** конфігом, якщо грамотно використовувати `projects` в `playwright.config.*`.

Але Playwright дає право вибору — тож варто відштовхуватись від потреб у конкретному проєкті.
