# Debug your tests

Дебаг (від англійського "debugging") - це процес пошуку, виправлення та вирішення помилок або невідповідностей у програмному коді.

## Етапи дебагу

- **Виявлення помилки**: знаходимо проблему, яка призводить до неправильного результату.
- **Репродукція проблеми**: відтворюємо помилку стабільно.
- **Аналіз коду**: дивимось змінні, логіку, стек викликів.
- **Виправлення**: вносимо зміни.
- **Перевірка**: переконуємось, що проблема зникла і нічого не зламали.

## Debug via IDE

### WebStorm

1) Відкрий проект у WebStorm → **Run** → **Edit Configurations**.
2) Додай нову конфігурацію типу **npm** і вибери скрипт, який треба запускати.
3) Встанови **breakpoint** у тесті.
4) Запусти **Debug** (кнопка або hotkey, зазвичай F9).
5) Використовуй Debug Tool Window:
- step over / step into
- variables
- call stack
- debug console

> Важливо: переконайся, що конфіг запуску налаштований на правильне середовище (env vars, baseURL, браузер тощо).

### VS Code

Playwright має сильну інтеграцію з VS Code.

Загальні кроки:
1) Встанови Playwright та Playwright extension.
2) Використовуй кнопки запуску тестів у редакторі.
3) Для дебагу — **Debug test** (правою кнопкою на тесті), не забудь поставити breakpoints.

## Debug with Playwright Inspector

Playwright Inspector — GUI інструмент для налагодження тестів:
- покрокове виконання
- редагування локаторів в реальному часі
- logs / actions

### Запуск Inspector

```bash
npx playwright test --debug
```

### Пауза в конкретному місці

Додай у тест:

```ts
await page.pause();
```

> Важливо: `page.pause()` **не працює у headless**. Запускай у headed/debug режимі.
